# Гипотеза 6 - Как лучше всего моделировать сезонность?

## Постановка задачи

Исследуем, какая стратегия генерации признаков лучше всего передаёт
сезонность для глобальной модели CatBoost на месячных временных рядах.

Сравниваются шесть наборов признаков:

| Набор | Обычные лаги | Сезонные лаги | Календарные | Фурье |
|-------|:-----------:|:-------------:|:-----------:|:-----:|
| `lags_only` | + | | | |
| `lags_seasonal` | + | + | | |
| `lags_calendar` | + | | + | |
| `lags_fourier` | + | | | + |
| `lags_seasonal_calendar` | + | + | + | |
| `lags_seasonal_fourier` | + | + | | + |

Бейзлайны: Naive, Seasonal Naive, AutoTheta, AutoETS.

## Данные

150 рядов из M4 Monthly, отобранных с балансировкой по силе сезонности
(слабая / средняя / сильная). Минимальная длина ряда - 72 наблюдения.
Горизонт прогнозирования - 18 шагов (стандарт M4 Monthly).

## Метрики

* sMAPE - симметричная средняя абсолютная процентная ошибка
* MASE - средняя абсолютная масштабированная ошибка (знаменатель - сезонный наивный прогноз на обучающей выборке)

Результаты анализируются в разрезе:
- общего качества по всем рядам
- силы сезонности (weak / medium / strong)
- шага горизонта прогнозирования (1–18)

